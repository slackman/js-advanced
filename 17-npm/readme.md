package.json
    версия - 1.3.2:
        1 - мажорная
        3 - минорная (фичи)
        2 - фиксы
    ^ - означает, что разрешено устанавливать пакеты выше только минор и фикс (2.0.0 не подойдет)
    ~ -  означает, что разрешено устанавливать приблизительно эту версию только фикс (1.4.0 не подойдет)
    без ~ и ^ - будет установлена точно та версия, которая написана

package-lock.json
    описывает текущие зависимости после установки

node_modules
    содержит все установленные зависимости. Хранятся в плоской структуре

команда npm ci восстановит установленные ранее зависимости из кэша
т.е если не хочется обновлять на новые версии зависимости в случае npm i,
то рекомендуется npm ci

npm audit fix --force не учитывает установленные ограничения в package.json,
жестко их перезапишет

npm dedup - проверяет пакеты на наличие дубликатов и удаляет их
npm cache verify - верифицирует кеш, если с ним какие-то проблемы
npm uni <package_name> - удалит зависимость. Либо можно просто удалить из package.json 

npm i -g - установка зависимости глобально
npm i -D - установка как dev dependency (для разработки)
    они устанавливаются в node_modules/.bin/
    команду для запуска можно написать в package.json - "scripts": {"build": "rollup"}

Dev Dependencies: Зависимости, необходимые только для разработки. Устанавливаются с npm install <package_name> --save-dev.
Global Dependencies: Устанавливаются глобально для доступа из любой точки системы с npm install -g <package_name>.
Peer Dependencies: Указывают на необходимость наличия другого пакета для работы текущего.
Optional Dependencies: Не критичные для работы проекта зависимости.
